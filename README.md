# GeoFleet Control (demo)

Демо-SPA **«моніторинг флоту / дронів»** на карті: маршрутизація, жива таблиця, карта (MapLibre), експорт у Excel. Розраховано на статичний хостинг (наприклад **GitHub Pages**).

## Стек

- **React 18** + **TypeScript** + **Vite**
- **react-router-dom** — сторінки: Dashboard, Map, Reports
- **Bootstrap 5** (CSS) — верстка
- **MapLibre GL** — растрові/векторні стилі, шари точок і лінії маршруту
- **react-virtuoso** — віртуалізований список на Dashboard
- **ExcelJS** — генерація `.xlsx` на клієнті (Reports)

## Архітектура даних (демо)

- **`FleetProvider` + `FleetContext`** — один екземпляр **Web Worker** (`demoFleet.worker.ts`) на всю сесію; оновлення флоту приходять повідомленнями `tick`.
- **Pending config**: у UI можна змінювати «бажану» кількість дронів і частоту тиків; застосування до воркера через **Start / Apply** (щоб слайдери не перезапускали симуляцію на кожному кроці).
- **`updatesById`** — останній знімок по `id` (для попапа й маршруту); **`updates`** у списку можуть фільтруватися за увімкненими типами на карті.

## Що реалізовано в проєкті

### Dashboard (`/`)

- Живий **віртуалізований** список дронів з того ж стріму, що й карта.
- **Фільтри** за типом (FPV, Bomber, drone_type_1, drone_type_2) і **регіоном**.
- **Клік по рядку** → перехід на **Map** з уже **виділеним** дроном (відкривається попап).
- **Акордеон «Як користуватись»** — короткі інструкції по Dashboard, Map, Reports і блок **«Про створення коду»** (використання ШІ).
- Карточка **«Що демонструє проєкт»** — перелік можливостей.

### Map (`/map`)

- Шар точок (колір за типом), клік по точці — **попап** з деталями (статус, швидкість, регіон, час польоту, витривалість тощо).
- **Маршрут**: для частини дронів (~40%, випадково після старту симуляції) воркер додає **синтетичний** `routePolyline`; кнопка в попапі / у панелі показує або ховає **жовту лінію** на карті. Якщо треку немає — кнопка disabled + короткий `title`.
- **Follow selected**, легенда з чекбоксами типів, вибір стилю карти (включно з кастомним випадаючим меню через проблеми з `<select>` поверх WebGL).
- Оновлення геоданих **батчаться в `requestAnimationFrame`**; позиція попапа й текст «живого» блоку **throttle** (~320 ms), щоб кнопка «Маршрут» не перестворювалась щотик і була зручна для кліку.
- **Start / Apply / Stop** — керування симуляцією і застосуванням конфігурації.

### Reports (`/reports`)

- Кнопка завантаження **`.xlsx`**: багато рядків, формули діапазону, окремий аркуш-підсумок (для демонстрації ExcelJS).

### Worker (`src/demo/realtime/demoFleet.worker.ts`)

- Спрощена **2D-симуляція** руху (коридори спавну, рух у бік «центру» сценарію, стани missing тощо).
- Для частини одиниць генерується **уявний минулий маршрут** (~10–15 хв модельного часу); остання точка лінії оновлюється разом із позицією.

## ШІ у створенні репозиторію

**Значна частина** коду та документації (компоненти React, воркер, MapPage, Dashboard, Reports, README, підказки UI) створювалась і ітеративно змінювалась за допомогою **великих мовних моделей** у середовищі на кшталт **Cursor**, з перевіркою людиною (`tsc`, лінтер, ручний перегляд). Це не приховується: див. також акордеон на Dashboard.

## Запуск локально

```bash
npm i
npm run dev
```

Збірка:

```bash
npm run build
```

## GitHub Pages

У `vite.config.ts` задано `base: './'`, тому вивід у `dist/` можна викладати на GitHub Pages без жорсткого шляху до імені репозиторію.

Типово: `npm run build` → завантажити вміст `dist/` (Actions або вручну).
